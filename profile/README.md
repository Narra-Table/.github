## 故桌是什么？

故桌是一款为 **TRPG 跑团** 设计的协作叙事平台。

它不是一个普通的聊天室，也不是功能堆砌的工具集合——而是一个让 GM 和 PL 都能专注在故事本身的工作台。

如果你试过在 QQ 里跑团、用骰娘掷骰、把模组存在云文档里、秘密信息靠私聊传递……那你应该明白我们想解决的是什么。

## 为什么选择故桌？

**以角色身份发言**

不用再说「我扮演的角色说」。选好面具，发言即角色。GM 自由切换 NPC、旁白、系统身份，一人多面。

**秘密信息不再靠私聊**

帷幕即消息的可见性开关。线索投放、单人剧情、GM 隐藏信息——谁该看到，谁不该看到，你说了算。

**聊天是过程，文档是沉淀**

跑团过程本身就是创作。消息可修订、可整理、可回顾。结团之后，整场冒险的故事依然完整可读。

**骰点与指令，原生支持**

告别第三方骰娘。掷骰、查询、战斗轮管理——需要什么，敲进去就行。

**桌面跑团，移动端也能跟**

GM 用桌面端备团控场，PL 用手机也能顺畅参与。随地开团，随时上车。

## 适合谁用？

- **GM（主持人）** — 备团、控场、管理角色与信息、记录与导出
- **PL（玩家）** — 沉浸扮演、掷骰查看、移动端参与、结团回顾


## 路线图

- **Phase 1 · 可用** — 开团、跑团、结团的基础闭环
- **Phase 2 · 好用** — 消息整理、线索系统、结团报告
- **Phase 3 · 可扩展** — 模组标准化、工具接入、自部署

## 加入我们

**故桌是一个学习型项目。** 它诞生于对 TRPG 的热爱，也成长于开放协作的过程中。

如果你是：
- **想积累实战经验的学生** — 准企业级开发流程（PR Review / CI / 规范提交）、前后端分离架构、React + RN 跨端开发，覆盖完整的工程化实践
